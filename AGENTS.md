# OpenClaw — Codex Instructions

## After Making Any Changes

After modifying any file in this repo, run:

```bash
./update-readme.sh
```

This regenerates `README.md` from the current state of `docker-compose.yml` and `data/openclaw.json`. Always keep README.md in sync.

## Repo Structure

```
openclaw/
├── docker-compose.yml       # Ports, env vars, API keys, volumes
├── server.mjs               # Patched proxy — auto-auth, fixed port 48728
├── update-readme.sh         # Regenerates README.md from current config
├── README.md                # Auto-generated — do not edit manually
├── AGENTS.md                # This file
├── data/
│   ├── openclaw.json        # Runtime config: models, gateway, plugins
│   ├── workspace/           # Agent workspace
│   ├── agents/              # Agent state and sessions
│   ├── credentials/         # API credentials (restricted)
│   └── identity/            # Device identity
├── skills/                  # Custom skills mounted into container
├── .env                     # Local secrets (not committed)
└── .kalshi_private_key.pem  # Kalshi private key (not committed)
```

## Key Files to Know

- **Add/change API keys** → `docker-compose.yml` under `environment`, then `docker compose restart`
- **Change default model** → `data/openclaw.json` at `agents.defaults.model.primary`, then `docker compose restart`
- **Enable/disable plugins** → `data/openclaw.json` under `plugins.entries`
- **Add custom skills** → drop files into `skills/`

## Docker Commands

```bash
docker compose up -d       # start
docker compose down        # stop
docker compose restart     # restart
docker compose logs -f     # follow logs
```

## Rules

- Do not edit `README.md` directly — it is auto-generated by `update-readme.sh`
- Do not commit `.env` or `.kalshi_private_key.pem`
- Keep `server.mjs` mounted via docker-compose volume — it patches auto-auth
